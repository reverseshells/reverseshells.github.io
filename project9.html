Cryptographic Decryption and Data Recovery Report

Introduction


As a security analyst, the ability to decrypt files and recover data is crucial for understanding and mitigating potential security threats. In this lab activity, we were presented with an encrypted home directory and tasked with deciphering the contents using Linux commands. The primary goal was to unveil hidden messages encrypted using the Caesar cipher.

Task 1: Reading Contents of a File


I began by exploring the home directory and examining the contents of a file named README.txt. The README.txt file provided essential instructions, guiding me towards a hidden file in the 'caesar' subdirectory. This hidden file, encrypted using the Caesar cipher, contained further instructions for data recovery

Task 2: Finding a Hidden File


Upon navigating to the 'caesar' subdirectory, i uncovered a hidden file named '.leftShift3.' This file employed a Caesar cipher with a left shift of three letters. Leveraging the 'tr' command, i successfully decrypted the hidden message, revealing a command for decrypting the encrypted data file in the next task.

Task 3: Decrypting a File


In the final task, armed with the decryption command provided in the '.leftShift3' file, i applied it to decrypt the 'Q1.encrypted' file in our home directory. The command utilized the OpenSSL tool, employing AES-256-CBC encryption with enhanced security through the -pbkdf2 option. The decrypted file, 'Q1.recovered,' was successfully generated.
