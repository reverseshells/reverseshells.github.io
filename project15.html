Cybersecurity Incident Report: SYN flood attack

Section 1: Identify the type of attack:


⦁ One potential explanation for the website's connection timeout error message is: ⦁ This event could be:
⦁ A network-level denial of service (DoS) attack , specifically a SYN flood attack.

Section 2: Explain how the attack is causing the website to malfunction:


⦁ Three-way Handshake Steps:
⦁ [SYN] (log items 52, 58, 69):
⦁ Initial request from an employee visitor to connect to the web server.
[SYN, ACK] (log items 53, 56, 58):
⦁ Web server's response agreeing to the connection, reserving resources.
[ACK] (log items 58, 60, 62):
⦁ The visitor's machine acknowledges the permission to connect.

Impact:


⦁ Malicious Actor's SYN Flood Attack:
⦁ Sending a large number of SYN packets (highlighted red in the logs).
⦁ Normal SYN requests (highlighted green) get responses initially.
⦁ Logs (items 125 onwards) show the server stops responding to legitimate traffic.
⦁ Logs Indication:
⦁ Abnormal number of SYN requests (red highlighting).
⦁ Yellow highlighted rows indicate failed communications between legitimate visitors and the server.
⦁ Errors in the logs include HTTP/1.1 504 Gateway Time-out and [RST, ACK] packets.
⦁ Impact on Server:
⦁ Overwhelmed by the SYN flood attack.
⦁ Unable to respond to legitimate connection attempts.
⦁ Network-level denial of service.

Recommendations:


⦁ Immediate Action:
⦁ Server temporarily taken offline for recovery.
⦁ Firewall configured to block the IP address responsible for the abnormal SYN requests.
⦁ Next Steps:
⦁ Short-Term:
⦁ Monitoring and adjusting firewall rules.
⦁ Regularly checking and blocking suspicious IP addresses.
⦁ Long-Term:
⦁ Implementing rate limiting to prevent SYN flood attacks.
⦁ Collaborating with the network team to enhance DDoS protection.
⦁ Next Generation Firewall (NGFW) performs deep packet inspection and protection against DOS and DDOS attacks
